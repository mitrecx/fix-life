# weekly summary 自动总结功能需求文档

## 1. 功能概述

### 1.1 功能简介
实现基于 Celery 的定时任务系统，每周一早上 5:00 自动生成用户上一周的日计划完成情况总结报告。该功能旨在帮助用户回顾每周任务完成情况，提升自我管理能力。

### 1.2 核心价值
- **自动化回顾**: 无需手动统计，系统自动生成周总结
- **持续改进**: 通过定期回顾帮助用户发现问题和改进点


## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 自动生成周总结
**触发条件**: 每周一早上 5:00（北京时间）

**处理流程**:
1. 系统扫描所有有日计划数据的活跃用户
2. 计算上一周的时间范围（上周一 00:00:00 - 上周日 23:59:59）
3. 查询该用户在上周时间范围内的所有日计划
4. 统计分析数据，生成总结报告
5. 保存总结报告到数据库
6. 发送通知提醒用户查看

**错误处理**:
- 如果某用户上周没有日计划数据，跳过该用户
- 记录生成失败的日志，不影响其他用户
- 支持手动触发生成（用于补救）

#### 2.1.2 周总结数据内容

**基础信息**:
- 周次标识（年份 + 周数）
- 时间范围（起始日期 - 结束日期）
示例: 2026年, 第 9 周(3月2日-3月8日)

**每天计划信息**:
- 每天的计划信息, 完成状态, 每日总结(如果有的话要显示)

**任务完成情况**:
- 已完成任务数量
- 未完成任务数量
- 整体完成率（百分比）


## 3. 技术实现方案

### 3.1 后端技术栈

**核心组件**:
- **Celery 5.x**: 分布式任务队列
- **Celery Beat**: 定时任务调度器
- **Redis**: 消息代理和结果存储
- **PostgreSQL**: 数据持久化
